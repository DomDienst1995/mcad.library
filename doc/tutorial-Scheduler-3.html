Tutorial: Scheduler-3

Scheduler-3

This tutorial demonstrates the handling of onQueue and onAnim events by attaching appropriate event handlers and using the timing information contained within the event to queue up a sound for playback and modify HTML elements in real-time in response to events.

To run this program, copy the code in Listing 1 into a new HTML file and copy the MCAD Pattern Scheduler and jQuery javascript files into the same folder. The program will produce output in the form of a metronome ticking for each 1/16th note whilst modifying tabular data in real-time in response to events. Opening up the console window (Ctl-Shift-J in Google Chrome) will produce log out put as in Figure 1 below.

Code Breakdown

With this example, the scheduler is sending out pulses in the form of events for every step as it is due to be queued for playback and as the step is actually played back in real-time. By attaching our event handler to the onQueue and onAnim events, our handlers are called every time a step is ready to be queued or when it is played back. The onQueue handler has one argument (in this instance it has been called step, but can be called anything), that contains both the timing and pattern stamps of the step to be queued whereas the onAnim handler has two arguments of type StepStamp representing the current (and previous) step position to be animated.

The onQueue event handler queues up an oscillator for playback in the same manner as the previous tutorial but also uses jQuery to modify the onQueue table column to the step stamp of the step in the onQueue event.

The onAnim event handler modifies the other column with this event's current step stamp in the same fashion as the onQueue handler. However, note that the onQueue and onAnim stamps are out of sync. This is because the onQueue events are in the future whereas the onAnim events are fired in real-time as the step is played back (the lookAheadTime and scheduleAheadTime options have been exaggerated to pronounce this effect). This means that the onQueue events cannot be relied on for real-time animation of step playback, hence the inclusion of the onAnim timing pulse.

			// Event handler for scheduler's onQueue events
			function playStep(step) {

				// Create an oscillator node and connect it to the context's output
				var oscillator = context.createOscillator();
				oscillator.connect(context.destination);

				// Queue osillator to start playback on step's playback time
				oscillator.start(step.time.swing);

				// If the step number has a remainder when divided by 4 (modulus operator: %), play 400 hz
				// Otherwise (on beat), play 800 hz
				if(step.stamp.step % 4)oscillator.frequency.value = 400;
				else oscillator.frequency.value = 800;

				// Queue oscillator to stop playback 100ms after step's playback time
				oscillator.stop(step.time.swing + 0.1); 
				
				// Modify the onQueue column in the event table to display this event's stamp information
				$("#qBar").text(step.stamp.bar);
				$("#qBeat").text(step.stamp.beat);
				$("#qStep").text(step.stamp.step);
				$("#qPattern").text(step.stamp.patternPos);
				$("#qGUID").text(step.stamp.guid);
			}
			
			// Event handler for scheduler's onAnim events
			function animateStep(currentStep, lastStep) {

				// Modify the onAnim column in the event table to display this event's stamp information
				$("#aBar").text(currentStep.bar);
				$("#aBeat").text(currentStep.beat);
				$("#aStep").text(currentStep.step);
				$("#aPattern").text(currentStep.patternPos);
				$("#aGUID").text(currentStep.guid);
			}
		
			// Attempt to create a Web Audio context, otherwise fallback on the older Webkit Audio context
			var context = new (window.AudioContext || window.webkitAudioContext)();
			
			// Create a scheduler with a tempo of 60 bpm in 4/4 time with 1/16th note steps and queue and animation event handlers
			var scheduler = new Scheduler(context, {tempo: 60, onQueue: playStep, onAnim: animateStep, scheduleAheadTime: 0.5, lookAheadTime: 0.25});
			
			// Initiate the scheduler's timing pulses
			scheduler.start();

		</script>
