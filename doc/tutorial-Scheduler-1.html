Tutorial: Scheduler-1

Scheduler-1

This tutorial demonstrates the creation of a scheduling object with a tempo of 128 bpm. Note that no other parameters have been specified, instead the defaults have been used (4/4 time, 16th notes, no event handlers, etc.).

To run this program, copy the code in Listing 1 into a new HTML file and copy the MCAD Pattern Scheduler javascript file into the same folder. The program itself will not produce any output, however opening up the console window (Ctl-Shift-J in Google Chrome) will produce log out put as in Figure 1 below.

Code Breakdown

The scheduler object is created using the appropriate constructor and supplying the Web Audio API audio context along with the (optional) configuration parameters. The configuration parameters are always placed between two squiggly braces with the comma separated parameter/value pairs. Any parameter not specified by the user will be set to the default value specified in the scheduler constructor.
Once the scheduling object has been created, the scheduler can be started by calling the start method. This will initiate the timing pulses for firing the onQueue, onAnim and onTween events.

Code Listing

<!doctype html>
<html>
	<head>
		<meta charset="utf-8">
		<!-- Include the MCAD Sampler class (make sure to copy to your project folder) -->
		<script src="mcad.pattern.scheduler.v1-0.js"></script>
		<!-- Add some basic styling to make the page look prettier -->
		<style>
			body {
				background-color: rgb(245, 245, 245);
				color: rgb(121, 121, 121);
				font-family: 'Droid Sans', Arial, sans-serif;
				font-size: 1em;
			}
			
			h1 {
				text-align: center;
			}
		</style>
		<!-- Page title (appears in browser/tab titlebar) -->
		<title>MCAD Scheduler Tutorial 1</title>
	</head>
	<body>
		<!-- Our HTML goes here -->
		<h1>A Simple Scheduler</h1>
		<p>Creates and initiates a scheduler with a tempo of 128 bpm in 4/4 time with 1/16<sup>th</sup> note steps.</p>
		<p>If running in Google Chrome, open the Developer Tools (Ctl-Shift-J) and view the log output.</p>
		<script>
		
			// Attempt to create a Web Audio context, otherwise fallback on the older Webkit Audio context
			var context = new (window.AudioContext || window.webkitAudioContext)();
			
			// Create a scheduler with a tempo of 128 bpm in 4/4 time with 1/16th note steps
			var scheduler = new Scheduler(context, {tempo: 128});
			
			// Initiate the scheduler's timing pulses
			scheduler.start();

		</script>
	</body>
</html>
